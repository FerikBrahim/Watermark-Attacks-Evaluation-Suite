# Watermark Attacks Evaluation Suite

## Project Title

**Watermark Attacks Evaluation Suite for Healthcare-Oriented Biometric Watermarking**

## Description

This repository contains the Python implementation used to evaluate the robustness of the biometric watermarking framework presented in the paper:

**“Healthcare-Oriented Biometric Watermarking Framework Using FlexenTech and Logistic Maps.”**

The suite applies compression, noise, filtering, geometric, resampling, and photometric attacks to watermarked images. It also calculates image-quality and watermark-recovery metrics to assess the resilience of the proposed approach under different image-processing conditions.

## Repository Structure

```text
Watermark-Attacks-Evaluation-Suite/
├── Attacks_01.py             # Basic image-processing attacks
├── Attacks_02.py             # Compression, geometric, noise, and filtering attacks
├── Attacks_03.py             # Extended attack-evaluation suite
├── Attacks_4.py              # Additional attack configurations
├── Missing_Attacks.py        # Histogram, blur, brightness, gamma, and JPEG2000 attacks
├── Metrics.py                # PSNR, SSIM, and NCC calculations
├── Metrics_Final.py          # BER, MSE, PSNR, SSIM, and NCC calculations
├── Watermarking.py           # Watermarking evaluation functions
├── attacked_images/          # Generated attacked images
└── attacked_images_missing/  # Additional generated attack results
```

## Usage Instructions

### 1. Prepare the Watermarked Image

Generate a watermarked image using the main implementation:

```text
https://github.com/FerikBrahim/FlexenTech-Biometric-Watermarking
```

Update the input-image path in the selected attack script.

### 2. Run the Extended Attack Suite

In `Attacks_03.py`, set:

```python
INPUT_IMG = "path/to/watermarked_image.png"
OUT_DIR = "attacked_images"
```

Run:

```bash
python Attacks_03.py
```

The script applies a comprehensive set of attacks, including:

- JPEG and JPEG2000 compression;
- additive Gaussian noise;
- salt-and-pepper noise;
- median filtering;
- Gaussian blur;
- sharpening;
- histogram and contrast modification;
- gamma correction;
- rescaling;
- rotation;
- cropping and padding;
- translation;
- affine transformation;
- color quantization;
- chroma subsampling;
- row and column loss; and
- re-watermarking.

### 3. Run Additional Attacks

Set the input-image path in `Missing_Attacks.py`, then run:

```bash
python Missing_Attacks.py
```

This script applies additional transformations, including:

- histogram equalization;
- motion blur;
- brightness shifting;
- gamma adjustment; and
- JPEG2000 compression.

### 4. Calculate Image-Quality Metrics

Set the image paths in `Metrics.py`:

```python
img1 = cv2.imread("watermarked_image.png")
img2 = cv2.imread("attacked_image.png")
```

Run:

```bash
python Metrics.py
```

The script reports:

- Peak Signal-to-Noise Ratio (PSNR);
- Structural Similarity Index (SSIM); and
- Normalized Cross-Correlation (NCC).

### 5. Calculate Watermark-Recovery Metrics

Set the original and extracted watermark inputs in `Metrics_Final.py`, then run:

```bash
python Metrics_Final.py
```

The script calculates:

- Bit Error Rate (BER);
- Mean Squared Error (MSE);
- Peak Signal-to-Noise Ratio (PSNR);
- Structural Similarity Index (SSIM); and
- Normalized Cross-Correlation (NCC).

## Evaluation Methodology

The robustness evaluation follows these steps:

1. generate a watermarked image using a fixed fingerprint payload and recorded embedding parameters;
2. save the original watermarked image;
3. apply one attack at a predefined severity level;
4. extract the watermark using the original embedding and extraction parameters;
5. compare the original and recovered watermark using BER and NCC;
6. compare the watermarked and attacked host images using PSNR and SSIM;
7. repeat stochastic attacks using fixed random seeds; and
8. record the attack type, severity, image dimensions, software versions, and evaluation results.

PSNR and SSIM measure the distortion introduced into the host image, whereas BER and NCC evaluate the accuracy of watermark recovery.

## Attack Categories

### Compression Attacks

- JPEG compression;
- JPEG2000 compression; and
- chroma subsampling.

### Noise Attacks

- additive Gaussian noise; and
- salt-and-pepper noise.

### Filtering Attacks

- Gaussian blur;
- median filtering;
- motion blur; and
- sharpening.

### Geometric Attacks

- rotation;
- affine transformation;
- translation;
- cropping;
- rescaling; and
- bending.

### Photometric Attacks

- brightness shifting;
- gamma correction;
- histogram equalization; and
- contrast modification.

## Reproducibility

For reproducible evaluation:

- use the same watermarked input image and watermark payload;
- preserve the embedding and extraction parameters used by the main implementation;
- specify the attack type and severity;
- use fixed random seeds for stochastic attacks;
- record the image dimensions and interpolation method;
- record the Python and library versions; and
- report the number of evaluated images and repeated trials.

A fixed random seed may be defined as follows:

```python
SEED = 2026
np.random.seed(SEED)
```
